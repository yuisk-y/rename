# 教学资料排序重命名工具

一个适用于 Windows 10/11 的本地桌面工具，用来给教学资料文件和文件夹按用户指定顺序批量添加序号。程序只读取用户选择目录的第一层对象，不递归处理子文件夹内容，不上传、不传输用户文件。

## 环境要求

- Windows 10/11
- Python 3.11 或以上版本
- PySide6
- pytest
- PyInstaller

## 安装依赖

```powershell
cd D:\aiworkspace\rename\file_sort_renamer
python -m pip install -r requirements.txt
```

## 启动程序

```powershell
cd D:\aiworkspace\rename\file_sort_renamer
python main.py
```

## 基本操作

1. 点击“选择文件夹”，选择教学资料所在目录。
2. 程序会显示该目录第一层中的文件和文件夹。
3. 在表格中按住整行拖动，可以调整章节顺序。
4. “参与”列可直接单击切换；取消勾选后，该对象会变灰并显示“已屏蔽”。
5. 被屏蔽对象不参与序号计算、不出现在确认映射中，也不会被重命名。
6. “排序标签”列可编辑。输入目标位置，例如 `1`、`2`、`30`，该条目会自动移动到对应顺序，适合大规模列表中避免长距离拖动。
7. 表格上方的搜索框用于定位对象；搜索不会过滤列表，只会滚动到命中项并高亮，按 Enter 或点击“下一个”可继续跳转。
8. 根据需要修改起始序号、序号位数、分隔格式，预览会立即更新。
9. 点击“一键重命名”，在确认窗口中检查完整的“原名称 → 新名称”映射。
10. 执行成功后，可点击“撤销上次重命名”恢复最近一次成功操作。

默认设置会生成类似 `01. 文件名.ext`、`02. 文件夹名` 的名称；把“序号位数”改为 `1` 可生成 `1. 文件名.ext`。

## 安全机制

- 重命名前检查目标名称重复、目标已存在、源对象不存在、权限风险、Windows 非法字符、Windows 保留名称、空名称和过长路径风险。
- 不覆盖任何已有文件或文件夹。
- 使用两阶段重命名：原名称 → UUID 临时名称 → 最终名称，可安全处理名称互换。
- 批量重命名失败时会尽力回滚已完成步骤，避免留下难以识别的临时名称。
- 最近一次成功重命名记录保存在程序自己的本地应用数据目录中，不会在用户资料文件夹内创建配置文件。
- 错误会写入程序日志，便于排查失败原因。

## 旧序号清理规则

勾选“删除原有序号”时，会保守清理以下常见前缀：

- `1. 消费者理论`
- `01、消费者理论`
- `01-消费者理论`
- `01_消费者理论`
- `第1章 消费者理论`
- `第01章-消费者理论`

不会仅仅因为名称以数字开头就删除数字，例如 `2026年度教学计划.docx`、`500强企业案例.pdf` 会保留原始数字。

## 已知限制

- 只读取所选目录第一层中的文件和文件夹，不递归显示子文件夹内部对象。
- 撤销只保存最近一次成功重命名操作。
- Windows 对超长路径、被其他程序占用的对象、权限继承等情况可能仍会在实际重命名时失败；程序会显示具体错误并尝试回滚。
- 重命名前会始终显示完整确认窗口，以降低误触风险。
